## Working defaults until these are resolved

- **Section 3 Q9 (resolved with caveats):** Total contributions use GT `analysis_total_contributions_amount` (990 Line 1h, 990-EZ Part I Line 1, and PF Part I Line 1). For Form 990 filers the contributions side is decomposed into six mutually exclusive segments via the Line 1 sub-channels: program service revenue, government grants received (1e), other institutional contributions (1a + 1d), likely-individual contributions (1b + 1c), the mixed Line 1f bucket, and residual revenue. The plan-language "Other contributions (foundation grants etc.)" segment maps to the institutional aggregate `analysis_calculated_grants_total_amount` = `FEDERACAMPAI` + `RELATEORGANI` + `GOVERNGRANTS` (Form 990 Lines 1a + 1d + 1e). Earlier versions of that aggregate accidentally summed `FOREGRANTOTA` and `GRANTOORORGA`, which are Form 990 Part IX (grants paid out) rather than contributions received; that mix has been removed. **Caveat:** Form 990 Line 1f mixes individual gifts with private foundation grants, DAF distributions, corporate gifts, and bequests in a single line that cannot be split without Schedule B. Form 990-EZ and 990-PF do not separately report Line 1 sub-components; their entire contributions total is routed into the mixed bucket. See OQ-008 for the unresolved follow-up about cleaner individual-versus-institutional separation.
- BMF stays the default count and density base.
- Postcard stays a supplement, not a silent count-base replacement.
- Efile stays the primary NCCS 990 source.
- Core is now a source-specific `2022` analysis layer, but it is not the default pooled base.
- GT stays the primary source for the current detailed revenue-source subcomponents.
